## 5) API 權限檢查清單（每次發版前）

- [ ] `POST /api/auth`：有寫入 `role` 到 JWT
- [ ] `POST /api/verify`：有回傳 `role`
- [ ] owner API：皆有 `requireRole(['owner','admin'])`
- [ ] employee API：不回傳敏感欄位
- [ ] Firestore 不允許 client 直讀敏感資料
- [ ] 角色測試帳號驗證通過（employee / owner / admin）

## 7) 對外展示摘要（可貼履歷/面試）

### 一句話版本

我主導建置零售 POS 智慧補貨系統，於同一 WebApp 實現多角色權限分流，並將成本與建議採購量安全上線，支援每日自動更新與報表匯出。

### 條列版本

- 以 Vercel API + Firebase 實作角色化資料存取（employee/owner/admin）
- 將 POS 同步、分類策略、建議採購量整合為每日運行流程
- 提供老闆決策介面：篩選、策略檢視、Excel 匯出、一鍵複製
- 建立可追溯機制（logic_version / as_of_date / Decision Log）
