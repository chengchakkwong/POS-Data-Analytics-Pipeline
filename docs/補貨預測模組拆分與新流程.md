# 補貨預測模組拆分與新流程（abc_xyz_analysis + inventory_forecast）

## 目的
把原本集中在單一腳本的補貨預測流程拆成兩段，讓責任分工更清楚、資料介面更穩定，並降低多進程運算時的記憶體/序列化壓力。

本流程的核心輸出改為「目標庫存」`Target_Stock`（不再輸出 `Suggested_Order / Final_Order`）。

---
## 模組與輸出介面

1. `abc_xyz_analysis.py`（分類/標籤）
   - 輸入：
     - `data/processed/DetailGoodsStockToday.csv`（今日庫存主檔）
     - `data/processed/vw_GoodsDailySales_partitioned`（歷史銷售）
   - 輸出：
     - `data/insights/abc_xyz_analysis.csv`
     - 主要欄位（供下游預測使用）：
       - `GoodsID`
       - `ABC_Class`（A/B/C/New/Excluded）
       - `XYZ_Class`（X/Y/Z/New，或 Excluded 的預設值）
       - `CV`（XYZ 變異係數）
       - `Mean_Monthly_Qty`
       - `Strategy`

2. `inventory_forecast.py`（目標庫存規劃）
   - 輸入：
     - `data/insights/abc_xyz_analysis.csv`
     - `data/processed/DetailGoodsStockToday.csv`
     - `data/processed/vw_GoodsDailySales_partitioned`
   - 輸出：
     - `data/insights/target_stock_plan.csv`
   - 輸出欄位（精簡且對齊下游）：
     - `ProductCode`, `Name`
     - `ABC_XYZ`（例如 `AX / BY / CZ`）
     - `Strategy`
     - `CurrStock`
     - `Base_Demand`, `Final_Demand`
     - `Target_Stock`

---
## 新品（New）判定與影響範圍
在 `abc_xyz_analysis.py` 中：
- `New` 的判定依據改為：商品「整段歷史」的首次上架日（`FirstSaleDate`）換算 `Month_Age`，並以 `Month_Age < 4` 作為新品判定。
- 新品一旦被標記為 `ABC_Class='New'`，在後續預測端會走新品策略（A/B/New 的預測範圍）。
- 注意：ABC（A/B/C）切分的價值彙總仍維持使用最近 12 個月的利潤計算邏輯；本次變更主要影響 `New` 的標記來源。

## ABC/XYZ 九宮格如何影響預測（目標庫存）

下游 `inventory_forecast.py` 的總體規則：
- 補貨計畫只處理 `ABC_Class` ∈ `{A, B, New}`，其餘 `C` 類走獨立分支
- `Excluded` 不會進入補貨計畫

### A/B/New：依 XYZ 分流到 X/Y（AI）或 Z（防禦規則）
在 `run_single_sku_forecast()`：

1. Base Demand（`base_pred`）由 ABC/XYZ 決定
   - `New`：
     - 以新品銷售期間的 run-rate（日均銷量）推 30 天需求
     - 若活躍天數不足（< 7 天），會對 `base_pred` 做上限夾住
   - `X`/`Y`：
     - 將月彙總序列轉成 `ds/y`
     - 月份數 >= 12：用 NeuralProphet（若可用）或 Prophet 做下一期預測
     - 月份數不足：用最近 3 期月銷平均當作 `base_pred`
   - `Z`：
     - 不跑時間序列模型
     - 用歷史月彙總的 last_6m `max`
     - 若最後 6 個月的最大值剛好等於歷史最大（且>0），視為可能存在離群大單，改取次大值

2. 類別季節加成（`boost`）
- `boost` 只在下列條件啟用：
  - `abc == 'New'` 或 `xyz == 'Z'`
- 加成來源：`calculate_category_seasonal_indices()` 產出的 `cat_index_map`（以 Category + 月份取平均季節係數）
- `boost` 上限 2.0

3. 安全庫存比例（`safety_ratio`）
- 用 `cv`（XYZ 變異係數）決定「安全墊」比例
- 上限規則：
  - `A`：最多 0.5
  - `B` / `New`：最多 0.3

4. 目標庫存上限防爆（非新品）
- 非 `New` 且 `mean_qty > 0` 時：
  - `target_stock <= max(mean_qty * 4, 2)`

因此九宮格的直觀對應是：
- `AX / AY`：X/Y 分支（AI 或最近3期均值）+ safety_ratio + 非新品上限
- `AZ`：Z 分支（防禦式 max/次大）+ boost + safety_ratio + 非新品上限
- `BX / BY`：同 AX/AY，但安全比例上限改為 B 的設定
- `BZ`：同 AZ，但安全比例上限改為 B 的設定
- `NewX / NewY / NewZ`：都走新品 run-rate；其中 `New` 會套用 `boost`（上限 2.0）

### C 類（CX/CY/CZ）：獨立分支（不看 XYZ）
在 `inventory_forecast.py` 的 C 分支中：
- `Mean_Monthly_Qty <= 0`：`Target_Stock = 0`
- `Mean_Monthly_Qty > 0`：
  - 若 `FirstOrderQty > 0`：
    - 若 `FirstOrderQty > Mean_Monthly_Qty * 12`：`Target_Stock = Mean_Monthly_Qty * 1.2`
    - 否則 `Target_Stock = FirstOrderQty`
  - 若沒有/無效 `FirstOrderQty`：`Target_Stock = Mean_Monthly_Qty * 1.2`
